Introducción

En este Workshop vamos a hablar sobre cómo ser más productivos escribiendo CSS.

Pero antes, deberías saber que:

Escribir CSS no es igual a cualquier lenguaje de programación.

  • No existen variables...
  • No se pueden hacer operaciones lógicas entre valores...
  • No posee flujos de control...
  • Esto es porque CSS es un Lenguaje de estilos.

Y por otro lado cada vez tenemos diseños más complejos:

  • Diseños dinámicos que cambian radicalmente por el tamaño de la pantalla... incluso por el tipo de dispositivo;

http://foundation.zurb.com/

  • Necesitamos sitios mejor optimizados, más ligeros, con mejores experiencias del usuario:

  • CSS también evoluciona (a sus pasos):

Sass y Compass nos van a dar una mano.


http://sass-lang.com/


http://compass-style.org/

¡Cuidado! Estadísticamente*, el 98% de las personas que escuchan hablar sobre estas herramientas, comentan en voz alta:

Escribir CSS es muy simple, no hace falta utilizar otras herramientas o aprender una nueva sintaxis.

¿Porqué utilizar estas herramientas?

  • Porque la simplicidad de CSS es su peor enemigo:
    • En sitios medianos a grandes, es dificil de organizar, escalar, mantener y refactorear...
  • Cada vez nos dedicamos más tiempo en investigación...
  • Más tiempo en escribir estilos...
  • Más tiempo en mantenimiento...

No somos máquinas, no nos interesa perder el tiempo, queremos ser más productivos.

Nivel 1: Novato

Charmander

¿Qué es Sass?

Sass es como escribir CSS en el 2025.
  • Creado en el 2007.
  • Desarrollado en Ruby (¡No requiere conocer el lenguaje!)
  • http://sass-lang.com/

Instalación de Sass (en 2 pasos)


Paso 1: Instalar Ruby y RubyGems

  • Windows: http://rubyinstaller.org/
  • Mac: Ya viene instalado :)
  • Linux: 
    # Debian
$ sudo apt-get install ruby1.9.1
$ sudo apt-get install rubygems

Paso 2: Abrir la consola/terminal y escribir:

$ gem install sass

Antes de empezar...

Sass maneja dos tipos de sintaxis:

Indentada (al estilo Ruby): 

/* estilos.sass */
#main
  color: blue
  font-size: 0.3em

La manera actual (al estilo CSS, y la que utilizaremos): 

/* estilos.scss */
#main {
  color: blue;
  font-size: 0.3em;
}

Comenzar a compilar

Abrir la consola, dirigirse a la carpeta en donde tenemos nuestros archivos .scss (navegar entre carpetas con el comando cd) y ejecutar:

$ cd <ruta_hacia_la_carpeta>
$ sass input.scss output.css

Sass leerá el archivo input.scss y lo compilará a output.css.

Es importante leer los mensajes de la consola, ya que en ella se muestran los cambios realizados, así como si surgen errores de compilación.

Vigilar por cambios

Si deseamos que el proceso sea automático, podemos hacer que Sass "vigile" si existen cambios en nuestros .scss:

$ sass --watch sass_folder:css_folder

Sass detecta cada cada vez que se realiza un cambio (incluso nuevas carpetas), compilando los archivos .scss y creando los nuevos .css dentro de la ruta indicada.

Cuando hayamos terminado el trabajo, para dejar que Sass siga ejecutandose, presionar Ctrl + C.

Convertir .css a .scss

En el caso que ya tengamos archivos .css creados, podemos renombrarlos manualmente a .scss, Sass los detectará sin problemas.

También se puede hacer el trabajo de manera automatizada con sass-convert:

$ sass-convert --from css --to scss --recursive <ruta>

Vamos a comenzar a ver las características más importantes de Sass.

Anidamiento

/* estilos.scss */	

// Anidamiento de selectores
.container{
  
  h3{
    color: blue;
  }
  
  p{
    // Anidamiento de estilos
    font: {
      size: 12px;
      weight: bold;
      family: sans-serif;
    }
  }
    
}

/* estilos.css */	

.container h3 {
  color: blue;
}
.container p {
  font-size: 12px;
  font-weight: bold;
  font-family: sans-serif;
}

Anidamiento

// Anidamiento de selectores
.container{
  
  a{
    text-decoration: underline;
    
    // Referencia al selector 
    // padre más cercano
    &:hover{
      text-decoration: none;
    }
	
    &:focus{
      color: red;
    }
	
    // Referencia al selector 
    // padre mayor
    .ie & {
      color: green;
    }
    
  }
    
}

.container a {
  text-decoration: underline;
}
.container a:hover {
  text-decoration: none;
}
.container a:focus {
  color: red;
}
.ie .container a {
  color: green;
}

Variables

// Variables globales
$amarillo: #F2FF62;
$ancho: 960px;

.container{
  width: $ancho;
  
  .header{
    // Utiliza $amarillo
    color: $amarillo;
    
    // Sobre-escribe la variable 
    // global $ancho
    $ancho: $ancho / 2;
    width: $ancho;
    
    // Define una nueva variable 
    // dentro de este scope
    $alto: 80px;
    height: $alto;
  }
  
  .footer{
    width: $ancho;
    //height: $alto; ¡Muestra error!
  }
  
}

.container {
  width: 960px;
}
.container .header {
  color: #f2ff62;
  width: 480px;
  height: 80px;
}
.container .footer {
  width: 480px;
}

Mixins

Los mixins son abstracciones que permiten la reutilización de estilos.

// Definición de un mixin
@mixin rounded-box($radio, $ancho : 100px){
  border-radius: $radio;
  -moz-border-radius: $radio;
  -webkit-border-radius: $radio;
  width: $ancho;
  height: auto;
}

.container{
  // Utiliza el mixin pasando 
  // dos argumentos
  @include rounded-box(15px, 960px);
  
  .box{
    // Utiliza el mixin pasando 
    // un solo argumento
    @include rounded-box(5px);
  }
  
  .box-left{
    // ¡Muestra error!
    // @include box-rounded(5px);
  }
  
}

.container {
  border-radius: 15px;
  -moz-border-radius: 15px;
  -webkit-border-radius: 15px;
  width: 960px;
  height: auto;
}
.container .box {
  border-radius: 5px;
  -moz-border-radius: 5px;
  -webkit-border-radius: 5px;
  width: 100px;
  height: auto;
}

Herencia de estilos entre selectores

.container{
  
  .box{
    border-radius: 5px;
    -moz-border-radius: 5px;
    -webkit-border-radius: 5px;
    width: 100px;
    height: auto;
  }
  
  .box-right{
    // Hereda los estilos de .box
    @extend .box;
    
    // Y luego añade los 
    // suyos propios
    border-right: 1px solid red;
  }
  
}

.container .box, .container .box-right {
  border-radius: 5px;
  -moz-border-radius: 5px;
  -webkit-border-radius: 5px;
  width: 100px;
  height: auto;
}
.container .box-right {
  border-right: 1px solid red;
}

Bonus 1: Funciones

De forma nativa, Sass contiene una gran variedad de funciones para realizar distintos calculos, por ejemplo:

$color: red;
$pi: 3.14159265;

h1{
  // Aclarece un 10% al rojo
  color: lighten(red, 10);
  
  &:hover{
    // Oscurece un 5% al rojo
    color: darken(red, 5);
    
    // Redondea a $pi
    line-height: round($pi) em;
  }
  
}

h1 {
  color: #ff3333;
}
h1:hover {
  color: #e60000;
  line-height: 3 em;
}

Más funciones: http://sass-lang.com/docs/yardoc/Sass/Script/Functions.html

También es posible definir nuestras propias funciones a través de @function:

// Calcular el Número áureo (Golden radio)

@function golden-radio($int) {
  
  // Devuelvo el número áureo
  @return floor($int / 1.62);
  
}

$width: 900px;

.container{
  width: $width;
  
  .left{
    width: golden-radio($width);
  }
  
  .right{
    width: ($width - golden-radio($width));
  }
  
}

.container {
  width: 900px;
}
.container .left {
  width: 555px;
}
.container .right {
  width: 345px;
}

Bonus 2: Estructuras de control y Bucles

Ideales para usar en conjunto con @mixins:

// Define un mixin
@mixin main-titles ($brand, $color: blue){
  
  // Dependiendo de la marca, define un valor
  @if $brand == google{
    $color : #3B56FF;
  } @else if $brand == facebook {
    $color : #76FF6B;
  }
  
  // Crea un bucle para los elementos h1 a h5
  @for $i from 1 through 5 {
    
    $amount : $i * 15;
    
    .#{$brand} h#{$i} {
      // Por cada vuelta, oscurece un 15%
      color : darken($color, $amount);      
    }
    
  }
  
}

// Llama al mixin definido
@include main-titles(google);

.google h1 {
  color: #0021ee;
}

.google h2 {
  color: #0016a1;
}

.google h3 {
  color: #000c55;
}

.google h4 {
  color: #000108;
}

.google h5 {
  color: black;
}

Estilos parciales

  • Archivos en donde su nombre comienza con un _guión bajo;
  • No se compilan a .css;
  • Ideales:
    • Pensar estructuras de estilos de manera modularizada;
    • Poder tener centralizadas variables / mixins que serán reutilizadas en varios lugares;
    • Poder mejorar la organización de estilos.

Estilos parciales

/* Archivo _variables.scss */
$ancho: 960px;
$azul: #3B56FF;

/* Archivo _header.scss */
.header{
  width: $ancho;
}

/* Archivo _footer.scss */
.footer{
  background: $azul;
}

/* Archivo _site.scss */
.site{
  width: $ancho / 2;
}

/* Archivo styles.scss (AKA pkg) */

// Importa cada estilo parcial
@import "variables";
@import "header";
@import "footer";
@import "site";

.header {
  width: 960px;
}

.footer {
  background: #3b56ff;
}

.site {
  width: 480px;
}

Estilos parciales

Otras maneras de usar @import:

// Si estamos importando un estilo parcial, 
// no es necesario usar el guión bajo al importar, 
// sin embargo Sass también lo detectará
@import "_variables";

// Lo mismo se puede hacer agregando la extensión del archivo
@import "header.scss";

// Y si se desea importar de la manera normal
// tan solo hay que usar la extensión .css
@import "footer.css";

// Incluso es posible realizar importaciones
// múltiples en una sola línea
@import "variables", "header", "footer", "site";

¿Y Compass?

Sass y Compass estan intimamente relacionados.
  • Compass es un framework que nace como una forma de poder aprovechar el potencial de Sass.
  • http://compass-style.org/
En su esencia, Compass no es más que un gran conjunto de estilos parciales, dividos en distintas categorias: CSS3, Utilidades, Helpers...

Instalación

$ gem install compass

Creación de un proyecto nuevo

$ compass create <miproyecto>

Compass genera automáticamente las carpetas para los archivos .scss / .css así como un archivo config.rb en donde se pueden indicar distintos parámetros de compilación y uso.

¿Y si quiero usarlo en un proyecto ya existente?

Basta con crear un archivo config.rb en la carpeta en donde vamos a trabajar:

#config.rb
css_dir = "css" # Carpeta en donde iran los .css
sass_dir = "sass" # Carpeta en donde iran los .scss
images_dir = "images" # Carpeta en donde van las imagenes
output_style = :expanded # :expanded, :nested, :compact, :compressed

Más info: http://compass-style.org/help/tutorials/configuration-reference/

Compilación usando Compass

En la carpeta en donde en donde se encuentra nuestro archivo config.rb:

$ compass compile

Compass lee el archivo de configuración para guiarse en donde buscar los archivos.

Y al igual que con Sass, también podemos hacer que "vigile" si existen cambios:

$ compass watch

CSS3

Utilizando Compass:

// Importo el módulo CSS3 de Compass
@import "compass/css3"; 

.box_gradient {
  // Sintaxis similar al estandar!
  @include background-image(linear-gradient(top, #444444, #999999)); 
}

.box_transition {
  @include transition(all 0.3s ease-out);
}

.box_shadow {
  @include box-shadow(0px 0px 4px #ffffff);
}

Sprites (AKA Adiós FireWorks...)

Paso 1: Tener las imágenes preparadas que se van a utilizar en el sprite

Paso 2: Crear un archivo .scss con:

// Indico que tome las imágenes .png que estan en /images/icons/
@import "icons/*.png";

// Crea el sprite
@include all-icons-sprites;

Notar dos cosas:

  • La ruta a donde ir a buscar las imágenes es relativa a la variable images_dir del config.rb.
  • El nombre del @mixin tiene que concordar con el nombre de la carpeta (en este caso "icons").

Sprites (AKA Adiós FireWorks...)

.icons-sprite, .icons-note, .icons-photo, .icons-script, .icons-tag {
  background: url('/images/icons-s0a4271b7ea.png') no-repeat;
}

.icons-note {
  background-position: 0 0;
}

.icons-photo {
  background-position: 0 -16px;
}

.icons-script {
  background-position: 0 -32px;
}

.icons-tag {
  background-position: 0 -48px;
}

Cada vez que agreguemos una nueva imagen, Compass se encargará de generar el sprite y volver a calcular las coordenadas de background-position.

Data URIs

Data URI es la especificación que permite mostrar determinada información con solo especificar el MIME type y la información misma encodeada. El formato básico es:

data:[<mime type>][;charset=<charset>][;base64],<encoded data>

Lo interesante es que podemos utilizarlo para "embeber" información dentro de nuestros CSS. Compass provee una función utilitaria que simplifica el proceso:

// Utilizando inline-image, Compass realiza 
// todo el proceso de conversión a Data URI
.logo{
  background-image: inline-image("logo.png");
}

// Estilo generado
.logo {
  background-image: url('data:image/png;base64,iVoAAABAA..<data>..gAAAAASII=');
}

Data URIs

El mayor beneficio de usar Data URIs es el ahorro de peticiones al servidor, ya que toda la información se encuentra dentro del mismo CSS.

Soporte:

  • Firefox 2+
  • Chrome
  • Safari
  • Internet Explorer 8+ (la información debe ser menor a 32KB)

Además no sólo se puede utilizar dentro del mismo CSS, sino tambien en el HTML:

Alternativas

Sass y Compass no son los únicos proyectos de su tipo:

  • LESS: Tiene una sintaxis parecida a Sass y mixins parecidos a los de Compass, pero utiliza JavaScript para el parseo de los estilos.
    http://lesscss.org/
  • Closure Stylesheets: Desarrollado por Google, utiliza un Jar para la generación de estilos.
    https://code.google.com/p/closure-stylesheets/
  • Bourbon: Esta basado en Sass, es una colección de @mixins con muchas similitudes a Compass.
    http://thoughtbot.com/bourbon/
